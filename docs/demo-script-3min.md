# Shufflow three-minute demo script

Target: 2:45–3:15 at 1440 × 900. Narration is approximately 400 words.

## 0:00–0:25 — proposition

**Visible:** Home, headline, complete Cube. **Action:** pause, then rotate the Cube or use the accessible fallback.

“Shufflow is an ADHD-first way to make progress without pretending attention always moves in a straight line. It breaks demanding work into manageable Pieces and lets us switch without losing our place. This complete Cube represents one Flow. I’ll rotate it to enter.”

## 0:25–0:45 — Flow Cube task map

**Visible:** `Finish PhD thesis chapter`, solid and outlined Pieces. **Action:** select the first `Empty Piece — create a task`.

“Inside the Flow, the Cube becomes the task map. Solid Pieces already contain work. The checked Piece is complete. Outlined Pieces are available space, so the screen stays calm instead of becoming another long task list. I’ll select an empty Piece.”

## 0:45–1:05 — capture

**Visible:** task workspace. **Action:** type `Write the validation methodology chapter`; press `Break down`.

“Task capture asks one question: what would I like to make progress on? I’ll add the validation methodology chapter. There are no categories or setup decisions in the way. Break down uses the existing AI workflow; Demo Mode has a deterministic fallback, so this moment is reliable even without a network key.”

## 1:05–1:30 — decomposition

**Visible:** six readable Steps and `Recommended first`. **Action:** briefly trace the list, then press `Start recommended Step`.

“One difficult Piece has now separated into six manageable Steps. The parent task remains clear, the Steps settle into a readable sequence, and Shufflow recommends a concrete starting point: Define the validation objectives. Starting is one direct action—there is no second Cube ritual or confirmation.”

## 1:30–2:10 — switching with context

**Visible:** focused Run. **Action:** dismiss the hint; swipe the open Run surface right, pause, then swipe left. Keep the gesture horizontal and begin away from the current Step and controls.

“Run keeps one current Step dominant, with a gentle timer and pause control. Swiping the task to the right Shuffles to another useful Piece: cleaning and summarising validation data. This is a supported attention shift, not a failure. Swiping left returns to the previous task with its Step and timer context preserved.”

## 2:10–2:35 — Step actions

**Visible:** current Step. **Action:** swipe the Step right to complete, then swipe the next Step left to skip. Fallback: `Place Piece`, then `Skip for now`.

“Gestures are scoped to the object under my pointer. A Step swiped right crosses into a completed state and updates its parent Piece. The next Step swiped left is skipped for now—not deleted or completed—so I can return later. Every gesture also has a labelled button and keyboard-accessible alternative.”

## 2:35–3:00 — reconstruction

**Visible:** Flow overview, then Progress. **Action:** choose `Exit`, then `Progress`.

“Back on the overview, the parent Piece shows the progress we just made. Progress reconstructs the same Flow Cube before showing precise supporting metrics. Separate Pieces become visible progress, even when the route through them was non-linear. Shufflow helps you work with your attention—and keep the whole Flow intact.”
